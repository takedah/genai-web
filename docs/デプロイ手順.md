# デプロイ手順

デプロイ手順のドキュメントは、閉域ネットワークへデプロイできるようにするため、デジタル庁 [genai-web](https://github.com/digital-go-jp/genai-web) を元に一部改変しています。ライセンスは CC BY 4.0 のもとで提供されています。

## 概要

本ドキュメントでは、パラメータ設定ファイルを使用したデプロイ方法について説明します。

## 1. パラメータ設定ファイルの準備

### 1-1. テンプレートファイルをコピー

```bash
cd packages/cdk/env-parameters
cp self-hosting-template.ts self-hosting-dev.ts
```

### 1-2. パラメータの設定

コピーした `self-hosting-dev.ts` を編集し、環境に合わせてパラメータを設定します。

- **必須項目**: `appEnv`（環境名）
- **任意項目**: `cdk.json` の `context` セクションに記載されている設定値（`allowedIpV4AddressRanges`、`modelIds`、`samlAuthEnabled` など）を、このパラメータファイルで設定してください。ファイル内の各コメントを参照しながら、必要に応じて変更してください。

### 1-3. デプロイ先リージョンの設定

cdk.context.json の AWS アカウント ID の箇所をデプロイ先のアカウントのものに修正し、リージョンもデプロイ先リージョンに修正してください。

```json
{
  "availability-zones:account=123456789012:region=ap-northeast-1": [
    "ap-northeast-1a",
    "ap-northeast-1c",
    "ap-northeast-1d"
  ]
}
```
## 2. parameter.ts の更新

`packages/cdk/parameter.ts` の `deploy_envs` オブジェクトに、作成した設定ファイルをインポートして追加します：

```typescript
import { selfHostingDevParams } from "./env-parameters/self-hosting-dev";

const deploy_envs: Record<string, Partial<StackInput>> = {
  "-selfHostingDev": selfHostingDevParams,
  // 他の環境も追加可能
};
```

> **Note:** `deploy_envs` に複数の環境を定義することで、`env=` パラメータでキーを指定して異なる環境へのデプロイが可能です。
>
> 例: `-c env=-selfHostingDev`、`-c env=-selfHostingProd` など

## 3. デプロイの実行

### ローカル環境からデプロイする場合

事前に Docker サービスを起動しておきます。

また、aws login コマンドなどで必要なクレデンシャルを取得しておきます。

```bash
npm -w packages/cdk run cdk -- deploy --all --require-approval never -c env=-selfHostingDev
```

上記コマンドの `-c env=-selfHostingDev` の部分で、`parameter.ts` の `deploy_envs` に定義したキーを指定することで、デプロイ先の環境を切り替えることができます。

### GitHub Actions を使用する場合

```bash
cp .github/genai-deploy.yaml.example .github/workflows/genai-deploy-{環境名}.yaml
```

上記コマンドでデプロイ用ワークフローファイルを作成後、ワークフロー内のコメントに従って環境名やブランチ名、リポジトリシークレットの設定をしてください。

## 4. 確認

デプロイ完了後、CloudFront の URL または設定したカスタムドメインにアクセスし、ログイン画面が表示されることを確認してください。
カスタムドメインの設定方法については、[カスタムドメイン設定](./カスタムドメイン設定.md) を参照してください。

## 補足事項

### パラメータの優先順位

- `cdk.json` 内の `context` は、デフォルト値として引き続き使用されます
- パラメータファイルで指定した値は、`cdk.json` の値よりも優先されます
- 複数環境（開発、ステージング、本番など）を管理する場合は、環境ごとにパラメータファイルを作成し、`parameter.ts` の `deploy_envs` に登録してください

### センシティブな情報の取り扱い

パラメータファイルには、API キー、認証情報、本番環境固有の設定などが含まれる場合があります。組織のセキュリティポリシーに従って、適切に管理してください。

特に本番環境の機密情報については、`.gitignore` への追加、または AWS Secrets Manager などの秘密管理サービスの使用を検討してください。

## トラブルシューティング

### デプロイエラーが発生する場合

- `appEnv` が正しく設定されているか確認してください
- パラメータファイルが `parameter.ts` に正しくインポートされているか確認してください
- `deploy_envs` のキー名と `-c env=` で指定している値が一致しているか確認してください
- AWS CLI が読み込む profile のリージョンが ap-northeast-1 に設定されているか確認してください
- cdk.context.json にデプロイ先リージョン以外のリージョンが指定されている場合は削除してから再デプロイを試みてください

### 環境切り替えがうまくいかない場合

- `parameter.ts` の `deploy_envs` に目的の環境が登録されていることを確認してください
- 登録されている環境のキー名と、デプロイコマンドで指定する `-c env=` の値が一致しているか確認してください
