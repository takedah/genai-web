# カスタムドメイン設定

## カスタムドメイン設定用ファイル

ドメイン関連の設定は `packages/cdk/parameter.ts` にて行います。  
以降は `parameter.ts` と表記します。

### ドメイン設定用変数

`parameter.ts` のドメイン関連設定用変数は、`stack-input.ts` で定義されている以下の 5 つのパラメータです。

- useHostedZone: Route53 の HostedZone を使うかどうか
- hostName: ホスト名（例: FQDNが `genai.digital.go.jp` なら `genai`）
- domainName: ドメイン名（例: FQDNが `genai.digital.go.jp` なら `digital.go.jp`）
- hostedZoneId: Route53 で作成した HostedZone の ID
- certificateArn: Amazon Certificate Manager の ARN

## カスタムドメインの設定が不要な場合

開発用など、CloudFront が割り当てるドメインで十分な場合は、`parameter.ts` のドメイン設定用変数の値をすべて空にしてください。

## ドメインをRoute53で管理可能な場合

Route53 でホストゾーンを作成してください。  
権威委譲を受けるために、ネームサーバーの値をドメイン管理者へ伝え、設定を依頼してください。

設定完了後、以下の値を設定してデプロイしてください。

- useHostedZone: true
- hostName（例: `genai`）
- domainName（例: `digital.go.jp`）
- hostedZoneId（作成したホストゾーンのID）

以上で、証明書の作成などが自動化され、CDK の管理下に置かれます。

## ドメイン管理を別サービスで行う場合

ここでは、ドメイン管理を別のサービスで行っており、Route53 に権威委譲できない場合の設定方法を説明します。

この場合は、まずカスタムドメイン設定を行わずに CDK で環境を構築してください。  
構築後、手動で Amazon Certificate Manager に証明書を作成してください。  
DNS 検証の場合は、証明書の発行後に CNAME 値を控えておいてください。  
次に、CloudFront のドメインを控えてください。  
その後、ドメイン管理者に、証明書の CNAME および CloudFront のドメインを CNAME 値として設定してもらうよう依頼してください。

### 設定例

| タイプ | ゾーン        | 名前、エイリアス、ホスト名等 | 値                                       | 用途                           |
| :----- | :------------ | :--------------------------- | :--------------------------------------- | :----------------------------- |
| CNAME  | digital.go.jp | \_hogehoge.genai             | _hogehoge_.fugafuga.acm-validations.aws. | ACMの証明書検証用              |
| CNAME  | digital.go.jp | genai                        | hogehoge.cloudfront.net                  | CloudFrontのカスタムドメイン用 |

ドメイン管理者による設定が完了すると、`Amazon Certificate Manager` の証明書が「発行済み」となり、カスタムドメインへアクセスできるようになります。

この設定後、CDK による管理下に置くため、`parameter.ts` の以下の値を設定してデプロイしてください。

- domainName（例: `digital.go.jp`）
- hostName（例: `genai`）
- certificateArn（`Amazon Certificate Manager`の証明書のARN値）
