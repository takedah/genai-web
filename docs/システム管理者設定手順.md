# システム管理者設定手順

## 概要

「管理者」には、「システム管理者」と「チーム管理者」の2つの種類があります。

- システム管理者（SystemAdminGroup）は、チームの作成およびすべてのチームの編集が可能です。
- チーム管理者（TeamAdminGroup）は、システム管理者によって設定され、特定チーム内のメンバーやアプリの追加・削除が可能です。

権限の詳細については、[チーム管理権限表](./チーム管理権限表.md)を参照してください。

ここでは「システム管理者」の設定方法を示します。

## スクリプトによる設定（推奨）

### 前提

- 対象環境のCloudFormationスタックがデプロイ済みであること
- 対象アカウントがCognitoユーザープールに登録済みであること（[アカウント登録手順](./アカウント登録.md)を参照）
- jq がインストール済みであること

### 手順

以下のスクリプトを実行してください。

```bash
# parameter.tsのenvキーは `'-dev': devParams,` となっていたらキー値（例では `-dev`）を指定してください
./scripts/add-system-admin.sh <parameter.tsのenvキー> <ユーザー名>

# 例
./scripts/add-system-admin.sh -dev user@example.com

# 成功した場合、以下のような出力が表示されます。
> 完了: ユーザー 'xxxxxxx@example.com' を SystemAdminGroup に追加しました。
```

## AWS コンソールによる手動設定

以下は、AWS コンソールから手動で設定する場合の手順です。

### 前提

- 対象環境の CloudFormation スタックがデプロイ済みであること
- 対象アカウントが Cognito ユーザープールに登録済みであること（[アカウント登録手順](./アカウント登録.md)を参照）

### 手順

1. Cognito ユーザープールの管理画面にアクセス
2. ユーザー一覧から管理者権限を付与したいユーザーを選択
3. グループ選択画面で `SystemAdminGroup` を選択

## SystemAdminGroup に追加後の確認

SystemAdminGroup への追加が完了すると、源内 Web のヘッダーの「アカウント」メニューに「チーム管理」メニューが表示されます。  
チーム管理にアクセスし、チーム作成や AI アプリの登録、メンバーの追加などが可能であることを確認してください。  

## トラブルシューティング

### チーム管理メニューが表示されない場合

- 対象ユーザーが SystemAdminGroup に正しく追加されているか確認してください
- 一度ログアウトしてから再度ログインしてください
